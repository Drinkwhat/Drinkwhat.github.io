Error Handling

Comprehensive error handling and exception management.

Overview

The error handling system is built around three components:

  • app/core/error_catalog.py — defines ErrorCode (enum), ErrorSeverity (enum), ErrorInfo (TypedDict), and the ERROR_CATALOG mapping

  • app/core/errors.py — defines AppError and typed subclasses; uses ErrorPayload and ErrorResponse TypedDicts for type-safe serialization

  • app/core/i18n.py — translates error messages (Italian default, English supported)

Exception Hierarchy

AppError (base)
├── AuthenticationError   (401 / 403)
├── ValidationError       (400 Bad Request)
├── BusinessLogicError    (404 / 409 / 422)
├── DatabaseError         (503 / 409 / 500)
├── ExternalServiceError  (503 / 504 / 502)
└── SystemError           (500 / 501)

Type Definitions

ErrorInfo (TypedDict)

Each entry in ERROR_CATALOG is a typed dictionary:

class ErrorInfo(TypedDict):
    message: str           # Italian message template (may contain {param} placeholders)
    severity: ErrorSeverity
    http_status: int
    action: str            # Recommended user action

ErrorPayload / ErrorResponse (TypedDict)

AppError.to_dict() returns an ErrorResponse, whose "error" key is an ErrorPayload:

class ErrorPayload(TypedDict):
    code: ErrorCode        # The ErrorCode enum value
    message: str           # Formatted, translated message
    severity: ErrorSeverity
    action: str
    details: str | None    # str(original_exc) or None

class ErrorResponse(TypedDict):
    error: ErrorPayload

Exception Classes

AppError (Base)

File: app/core/errors.py

class AppError(Exception):
    def __init__(
        self,
        code: ErrorCode,
        params: dict[str, Any] | None = None,
        original_exc: Exception | None = None,
        lang: str = "it",
    ):
        self.code = code
        self.params = params or {}
        self.original_exc = original_exc
        # ... (metadata from ERROR_CATALOG)

Purpose: Base for all custom exceptions with structured error codes

Usage:

raise AppError(
    ErrorCode.SYS_INTERNAL_ERROR,
    params={"request_id": "req-12345"}
)

ValidationError (400)

When: Input data is invalid

Error Codes:

  • VAL_INVALID_INPUT - Invalid input data

  • VAL_EMPTY_FILE - Empty file upload

  • VAL_FILE_TOO_LARGE - File exceeds max size (413)

  • VAL_NO_TEXT_FOUND - No text detected in image

  • VAL_IMAGE_ERROR - Image processing error

Examples:

  • Empty file upload

  • Invalid date format

  • Missing required field

  • No text found in image

Usage:

from app.core.errors import ValidationError, ErrorCode

if not file_bytes:
    raise ValidationError(ErrorCode.VAL_EMPTY_FILE)

if not text_annotations:
    raise ValidationError(ErrorCode.VAL_NO_TEXT_FOUND)

Response:

{
  "error": {
    "code": "ERR_VAL_004",
    "message": "Il file caricato è vuoto.",
    "severity": "warning",
    "action": "Carica un file valido non vuoto."
  }
}

ExternalServiceError (502)

When: A call to an external service fails (Vision, Gemini, other APIs)

Causes:

  • API quota exceeded

  • Invalid credentials

  • Network error

  • API temporarily unavailable

Usage:

try:
    text = ocr_service.extract_text(file_bytes)
except Exception as e:
    logger.error(f"Vision API failed: {e}")
    raise ExternalServiceError(
        ErrorCode.EXT_VISION_ERROR,
        params={"details": str(e)},
    )

Response:

{
  "error": {
    "code": "ERR_EXT_004",
    "message": "Errore nel servizio Google Vision: quota superata",
    "severity": "error",
    "action": "Verifica la qualità dell'immagine o riprova più tardi.",
    "details": "quota superata"
  },
  "request_id": "..."
}

When: LLM fails to parse data

try: response = client.document_text_detection(image=image) except Exception as exc: logger.error(f”Vision API call failed: {exc}”) raise ExternalServiceError( ErrorCode.EXT_VISION_ERROR, params={“details”: str(exc)}, original_exc=exc )

BusinessLogicError (404/409/422)

When: Business rule violations

Error Codes:

  • BUS_PRODUCT_NOT_FOUND - Product not in catalog (404)

  • BUS_RESOURCE_NOT_FOUND - Resource doesn’t exist (404)

  • BUS_ALREADY_EXISTS - Duplicate resource (409)

  • BUS_PARSING_ERROR - AI parsing failed (422)

  • BUS_OPERATION_NOT_ALLOWED - Operation not permitted (400)

Usage:

from app.core.errors import BusinessLogicError, ErrorCode

if not product:
    raise BusinessLogicError(
        ErrorCode.BUS_PRODUCT_NOT_FOUND,
        params={"product": product_name}
    )

Response:

{
  "error": {
    "code": "ERR_BUS_002",
    "message": "Prodotto 'Latte Intero' non trovato in questo store.",
    "severity": "warning",
    "action": "Aggiungi il prodotto all'anagrafica prima di procedere."
  }
}

DatabaseError (409/500)

When: Database operations fail

Error Codes:

  • DB_INTEGRITY_ERROR - Constraint violation (409)

  • DB_CONNECTION_ERROR - Connection failed (500)

  • DB_QUERY_ERROR - Query execution error (500)

Causes:

  • Duplicate records

  • Foreign key violations

  • Unique constraint violated

  • Connection issues

Usage:

from app.core.errors import DatabaseError, ErrorCode

try:
    db.add(product)
    db.commit()
except IntegrityError as exc:
    db.rollback()
    raise DatabaseError(
        ErrorCode.DB_INTEGRITY_ERROR,
        params={"details": str(exc)}
    )

Response:

Exception Handling Pattern

Try-Catch-Raise

from app.core.errors import ValidationError, DatabaseError, ErrorCode
from app.modules.auth.dependencies import AuthUser, Permission, RequirePermission

@router.post("/inventory/confirm")
async def confirm_batch(
    data: BatchConfirm,
    user: AuthUser = Depends(RequirePermission(Permission.INVENTORY_WRITE)),
    db: Session = Depends(get_db)
):
    # Validate input
    if not data.product_name:
        raise ValidationError(ErrorCode.VAL_MISSING_FIELD)

    try:
        # Convert date
        exp_dt = datetime.strptime(data.expiry_date, "%Y-%m-%d").date()
    except ValueError:
        raise ValidationError(ErrorCode.VAL_INVALID_FORMAT)

    try:
        # Database operation
        db.add(new_batch)
        db.commit()
    except IntegrityError as exc:
        db.rollback()
        raise DatabaseError(
            ErrorCode.DB_INTEGRITY_ERROR,
            params={"details": str(exc)}
        )
    except Exception as exc:
        db.rollback()
        logger.error(f"Database error: {exc}")
        raise SystemError(
            ErrorCode.SYS_INTERNAL_ERROR,
            original_exc=exc
        )

    return response

Best Practices

✅ Do

from app.core.errors import ValidationError, DatabaseError, ErrorCode

# Use specific error codes
raise ValidationError(
    ErrorCode.VAL_EMPTY_FILE,
    params={"details": "Product name cannot be empty"}
)

# Log before raising
logger.warning(f"Invalid input: {error}")
raise ValidationError(ErrorCode.VAL_INVALID_INPUT, params={"details": error})

# Re-raise with context and preserve original exception
except Exception as exc:
    logger.error(f"Database operation failed: {exc}")
    raise DatabaseError(
        ErrorCode.DB_INTEGRITY_ERROR,
        original_exc=exc
    )

❌ Don’t

# Generic exception without error code
raise Exception("Error")

# Don't expose internal details
raise AppError(ErrorCode.DB_QUERY_ERROR, params={"details": str(db_error)})  # ← Contains SQL details

# Don't log and ignore
try:
    db.commit()
except:
    pass  # ← Silent failure

# Don't catch too broadly without re-raising
try:
    ...
except:  # ← Catches everything
    ...

Fail-Open Architecture

This ensures high availability even when secondary infrastructure components fail.

Background Job Status

When monitoring background tasks via arq:

  • Mapping: The API maps internal arq statuses (e.g., in_progress, complete) to the JobStatus enum.

  • Failures: If a job finishes but the internal logic fails (e.g., AI parsing error), the result status is explicitly mapped to FAILED to provide clear feedback to the client.

  • Missing Jobs: If a job_id is expired or invalid, the system returns JobStatus.NOT_FOUND instead of failing, allowing the client to handle the state gracefully.

Internationalization (i18n)

Error messages support multiple languages. The default language is Italian (from ERROR_CATALOG). English is fully supported via app/core/i18n.py.

Coverage

All ErrorCode values have both an Italian entry in ERROR_CATALOG and an English translation in TRANSLATIONS["en"]. Regression tests in tests/unit/test_errors_system.py enforce this:

  • test_all_error_codes_have_catalog_entries — verifies every ErrorCode has an ERROR_CATALOG entry

  • test_all_error_codes_have_en_translations — verifies every ErrorCode has an English translation
